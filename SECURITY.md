# 安全政策

## 支持版本

当前仅维护最新公开版本。

## 报告问题

不要在公开 Issue 中上传真实敏感文档、清理输出、用户名、绝对路径、注册表导出或访问凭据。

如仓库已启用 GitHub Private Vulnerability Reporting，请优先使用该入口。否则可创建不含敏感材料的最小公开 Issue，并说明需要私下提供虚构复现样本。

报告应包含：

- 受影响的文件类型和版本；
- 选择的清理模式；
- 预期结果与实际残留；
- 使用虚构内容制作的最小样本或生成脚本；
- 是否能稳定复现。

元数据格式具有开放扩展能力。“未检测到已知元数据”不是绝对匿名保证；高敏感文件需人工复核。
